# 电工鼠实验室设计

## 当前架构分析

改造前的 `electric-mouse` 是一个原生 Canvas 课件：HTML 提供三栏布局，`main.js` 同时管理 10 个线性关卡、元件状态、图搜索、电流动画、输入和绘制。电路判定通过枚举电池两极间的简单路径完成，亮度按路径中的灯泡数量估算。该结构适合固定题目，但无法稳定支持大型混合网络、真实测量、编辑历史和版本化保存。

项目本身是无框架的纯静态站点。`src/electric-mouse/` 是源码，`npm run build` 使用 esbuild 压缩并输出到 `courseware/electric-mouse/`；GitHub Pages 与 Cloudflare Pages 发布同一份构建产物。画布方案继续使用 Canvas 2D，以保留现有夜间小镇视觉和大量导线粒子的绘制性能。

## 新架构设计

实验室拆成四层：

1. `circuit-model.js`：版本化文档、元件目录、端口、导线和增删 API。
2. `circuit-simulator.js`：与 UI 无关的确定性直流电阻网络求解和保护状态推进。
3. `circuit-storage.js`：校验、序列化、自动保存和手动存档。
4. `main.js`：Canvas 绘制、Pointer Events、触摸手势、编辑历史、属性与观察面板。

DOM 负责可访问按钮、抽屉、属性输入和实时读数；Canvas 只负责实验台、元件、导线、选择框和动态反馈。构建过程不需要新运行时或 CDN。

## 电路模型

`CircuitDocument` 是持久化和撤销的唯一事实来源。每个元件持有稳定 ID、世界坐标、旋转角、显式端口、可编辑 `properties` 和可持久化 `state`。导线只连接两个显式端口；导线交叉不会产生节点，接线柱的四个端口才会在模型中合并为同一电气节点。

第一版元件：

- 电源：电池、电池盒。
- 负载：灯泡、电动机、蜂鸣器。
- 控制：单刀开关、SPDT 双控开关、按钮开关。
- 控制与保护：可手动分断、过流跳闸并复位的总开关，以及带热量累计的保险丝。
- 连接：导线、四端接线柱。

电池盒作为可配置电压源处理；电动机和蜂鸣器使用电阻负载近似。灯泡可被拧下，保险丝熔断、断路器跳闸和电池保护都会把对应支路变成开路。

## 仿真方法

仿真先用并查集合并接线柱内部端口，再建立节点电导矩阵。每根导线作为 `0.005 Ω` 的数值支路参与求解，因此悬空导线不会误显示电流，也能得到各导线的常规电流方向：

- 电阻负载按 `G = 1 / R` 写入矩阵。
- 电池的开路电压与内阻使用等效 Norton 源写入矩阵。
- 断开的开关、被拧下或损坏的灯泡、熔断保险丝不写入导通边。
- 为孤立节点加入极小 `GMIN`，使断路网络仍可稳定求解。

高斯消元得到节点电位后，计算各元件的电压、电流和功率。灯泡亮度采用教育化映射：

`brightness = clamp(actualPower / ratedPower, 0, 1)`

保险丝、断路器和电池保护按离散时间累计热量：超过额定电流时升温，低于额定值时冷却，超过阈值后才熔断、跳闸或保护。暂停仿真时热量不继续累计。

## 交互状态机

画布只允许一个主交互状态：

- `idle`：命中测试和悬停。
- `placing`：已从工具箱选取元件，点击画布放置。
- `wiring`：端口按下后拖向目标端口，或第一次点击后等待第二个端口。
- `dragging-component`：拖动一个或多个已选元件。
- `marquee`：空白处拖出框选区域。
- `panning`：工具栏手掌按钮进入单指平移；桌面仍支持空格、辅助键和中键，触屏仍支持双指缩放和平移。
- `pinching`：双指缩放并保持手势中心对应的世界坐标不漂移。
- `context`：长按元件后显示操作、旋转、复制和删除菜单。

Pointer Events 与 `setPointerCapture` 保证快速拖动不丢失。按钮开关在按下时导通、松开或取消指针时断开。所有双击型操作都有观察面板和长按菜单入口。

## 已知限制

- 求解器只覆盖稳态直流电阻网络，不模拟交流、电感、电容、启动电流和真实导线参数；`0.005 Ω` 导线仅用于稳定求解和分辨支路电流。
- 电动机转速、蜂鸣器响度、灯泡发光和过压状态是可观察的教育化近似，不是器件数据手册模型。
- 保险丝、断路器和电池保护使用简化热量累计，不用于选择真实保护器件。
- 电池电量第一版仅显示和保存，不随能量消耗下降。
- 导线粒子按常规电流方向运动，不表示金属导体内电子的漂移方向或真实速度。
- 第一版使用单个本地手动存档槽，不提供云同步或多人协作。
