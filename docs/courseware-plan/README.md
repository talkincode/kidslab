# Primary School Interactive Courseware Master Plan

## 🎯 设计原则（每个课件都必须遵守）

1. **玩法即知识，不做题海换皮**
   知识点必须长在游戏机制里，而不是「答对题才能喂鱼」。孩子为了玩得更好而自然用到知识，输赢反馈本身就是概念反馈。禁止：弹窗讲解轰炸、答题解锁关卡、纯贴纸奖励驱动。

2. **每个课件要有一个「魔法时刻」**
   一个让孩子「哇！」出声、愿意拉爸妈来看的瞬间（如：亲手把水蒸气变成雪、发现镜子能拐弯激光、把地球转到让北京日出）。清单中每个条目都写明了这个时刻。

3. **低门槛，高上限**
   30 秒内上手，无需说明书；但留有可钻研的深度（最优解、自由创作模式、隐藏彩蛋）。1–2 年级课件做到 **零阅读门槛**：图标 + 语音提示 + 演示动画即可玩。

4. **引导而非灌输**
   允许失败、鼓励乱试，失败反馈要好笑而不是挫败（塌桥、爆锅、机器人把黄油整块塞进烤面包机）。孩子先形成直觉，术语最后才轻轻出现。

5. **双语 + 主题适配（工程硬要求）**
   - 全部界面文案提供 zh/en，读写 `localStorage: kidslab.lang` 与主站同步；
   - 低年级课件的双语优先靠图标和音效，文字仅作辅助；
   - 遵循主站深浅主题（变量见 `assets/css/app.css`）；
   - 课件完全独立、相对路径、第三方库走本地 vendor（three.js 参考 `src/welcome/vendor/`）。

## 🚦 优先级与建设批次

- **P0 首批（16 个）✅ 已全部上线**：知识点最核心 + 魔法时刻最强，四科、三年级段全覆盖，先做出平台气质。
  - 数学：✨ 数字萤火虫 · 🍎 果园订单 · 🐹 仓鼠围栏 · 🔭 数字宇宙 · 🎡 滚轮乐园
  - 编程：🍳 机器人早餐 · 🥁 循环乐队 · 🦁 如果动物园 · 🐧 排序运动会
  - 逻辑：🐛 规律毛毛虫 · 🕵️ 侦探线索板 · 🦊 狐狸的石子
  - 科学：☁️ 云朵工厂 · 🪩 分子迪斯科 · 🐭 电工鼠小镇 · 🌍 地球调度员
- **P1 第二批**：补齐各年级知识点主干。
- **P2 第三批**：锦上添花与长尾知识点。

每个条目的优先级已在学科清单中标注。批次内建议「一科一个」轮流上新，保持四个分类都有新鲜感。

## 🔧 技术选型速查

| 技术 | 适用 | 规划中使用的课件 |
|---|---|---|
| DOM/CSS 动画 | 卡片、拖拽、界面型玩法 | 大多数低年级课件 |
| Canvas 2D | 粒子、物理、绘图类 | 数字萤火虫、磁力冰球等 |
| **three.js**（本地 vendor） | 3D 观察、空间感、宇宙尺度 | 立体影子剧场、方块摄影棚、体积水族馆、数字宇宙、冰淇淋几何、地球调度员、月相狼历、星海水手、消化道漂流记 |
| WebAudio | 节奏、音高、听觉模式 | 循环乐队、声波乐团、分数节拍屋、规律毛毛虫 |

## ✅ 落地检查单（每个课件开工前过一遍）

- [ ] `src/<id>/` 目录名 = `course.json.id`，kebab-case
- [ ] `title`/`description` 双语齐全，`category` 与本清单一致，`levels` 含 `primary`
- [ ] 有明确的「魔法时刻」并在 30 秒内可触达
- [ ] 失败反馈有趣、可立即重试；无强制阅读长文案
- [ ] 深浅主题、语言切换、移动端触屏均可用
