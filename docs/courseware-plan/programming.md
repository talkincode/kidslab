# 编程课件清单 · Programming Courseware Plan

> 分类 `programming` ｜ 共 **14** 个 ｜ 已有课件：🐢 海龟画室 `turtle-studio`（顺序指令、循环、图形化编程入门）
> 覆盖：指令与顺序 → 循环 → 条件 → 变量 → 事件 → 函数 → 调试 → 算法 → 数据表示（二进制/像素/压缩/加密）→ 现代主题（机器学习/网络）

标注说明：`G年级` `优先级 P0>P1>P2` `技术`。每条含 **玩法** 与 **魔法时刻**。

### 1. 🍳 机器人早餐 · Robot Breakfast — `robot-breakfast`
`G1-2` `P0` `DOM 动画`｜顺序执行、指令的精确性、程序=步骤序列
- **玩法**：家里的机器人管家「完全按字面执行命令」。孩子用图标卡片排早餐流程：拿面包→放烤箱→涂黄油。顺序错了不报错——而是忠实演出灾难：先涂黄油再进烤箱，烤箱糊成一团黑烟；忘了「打开瓶盖」就倒果酱，机器人捏着瓶子干挤。每关目标是让挑剔的小主人吃上完整早餐。
- **魔法时刻**：第一次亲眼看到机器人把「没说清楚的话」执行成一场喜剧——「电脑不猜心思」从此刻进直觉。
- **进阶**：步数限制（逼出合并动作）、双机器人分工（并行萌芽）、录制自己的指令考爸妈。

### 2. 🥁 循环乐队 · Loop Band — `loop-band`
`G2-3` `P0` `WebAudio`｜循环、重复次数、嵌套循环
- **玩法**：组建动物乐队，但乐谱只能用「循环块」写：`重复4次[咚 哒]` 鼓手就永远打下去。想要「主歌×2 + 副歌×1 再整体×3」？把循环套进循环。孩子不是在学循环——是在编曲，循环只是让歌变长变酷的工具。写得越紧凑（同样的音乐用更少的块），乐队评分越高。
- **魔法时刻**：把 32 个零散音符块压缩成一个双层嵌套循环，音乐一模一样——「原来重复的规律可以被折叠」。
- **进阶**：三层嵌套做完整歌曲结构、错拍找 bug（循环次数差 1）、导出自己的循环神曲。

### 3. 🦁 如果动物园 · The If-Zoo — `iffy-zoo`
`G3-4` `P0` `DOM`｜条件判断 if/else、逻辑运算与/或/非、规则的边界情况
- **玩法**：动物园的自动喂食机由孩子编规则：「如果 是食草动物 → 投喂青草，否则 → 投喂肉」。动物排队涌来，规则机自动分流——直到熊猫出现：它是食肉目却吃竹子！孩子必须加特例规则、调整规则顺序（先判断特例再判断一般）。后期条件变复合：「如果 饿了 且 是夜行动物」。
- **魔法时刻**：看着自己写的一串规则像海关闸机一样自动处理了 30 只动物的早餐，一只都没喂错。
- **进阶**：规则条数限制（逼出优雅的 else）、神秘新物种压力测试、给淋浴水温写 if 规则（连续量条件）。

### 4. 🏥 虫虫医院 · Bug Hospital — `bug-hospital`
`G3-5` `P1` `DOM`｜调试、单步执行、阅读程序、预测输出
- **玩法**：坏掉的小程序是住院的病人（画歪的房子、数错的计数器、转圈不停的机器人）。孩子当医生：先「听诊」——单步执行程序，心电图式的执行指示灯一行行走，变量值在监视器上跳动；找到病灶行，开刀修改，病人当场康复出院（房子画正、机器人到站）。诊断优先于修改：先在病历上勾选「预测它会在哪一步出错」，猜对再动手加分。
- **魔法时刻**：第一次靠单步执行在第 7 行「抓到」bug 现行——原来程序出错的瞬间是可以亲眼看见的。
- **进阶**：疑难杂症（差一错误、条件写反、死循环）、只许看不许改的「读片」考试、限改一行的微创手术。

### 5. 🫙 巫师的罐子 · The Wizard's Jars — `wizard-jars`
`G4-5` `P1` `DOM`｜变量（命名、赋值、读取、更新）、值随程序变化
- **玩法**：巫师塔里所有魔法都靠贴着标签的罐子运转：`金币`、`血量`、`龙的愤怒值`。孩子写咒语脚本经营巫师塔：客人买药水 → `金币 = 金币 + 5`；喂龙 → `愤怒值 = 愤怒值 - 3`。罐子是透明的，每执行一行，罐里的量亲眼可见地涨落。核心谜题：同一个罐子在脚本不同时刻被读到不同的值——「变量是会变的量」不用解释。
- **魔法时刻**：写出 `愤怒值 = 愤怒值 + 愤怒值` 让龙瞬间狂暴翻倍，全塔警报——孩子笑着理解了「右边先算、左边接住」。
- **进阶**：两罐交换（经典 swap 谜题：需要第三个空罐！）、计数器与累加器、罐子命名太烂导致的事故关（可读性）。

### 6. 👻 惊喜鬼屋 · Surprise Haunted House — `haunted-events`
`G4-5` `P1` `DOM`｜事件驱动、触发-响应、事件与顺序程序的区别
- **玩法**：装修一座（可爱的）鬼屋：把「当…就…」导线接到机关上——「当 大门打开 → 蝙蝠群起飞」「当 有人踩到第三块地板 → 骷髅唱歌」。装好后按「营业」，AI 小游客进屋乱逛，你的机关在正确时机触发才能把他们逗得又叫又笑（惊吓值+爆笑值双计分）。孩子体会到：这程序没有「第一行」，一切看事件何时发生。
- **魔法时刻**：游客的一个动作触发连锁反应——蝙蝠撞铃铛、铃铛惊醒幽灵、幽灵打翻南瓜，整屋机关像多米诺一样活了。
- **进阶**：事件冲突调试（两个机关抢同一个触发器）、定时事件（午夜钟声）、开放日让别人的小人来逛你的鬼屋。

### 7. 🧁 函数厨房 · Function Kitchen — `function-kitchen`
`G5-6` `P1` `DOM`｜函数与封装、参数、复用、抽象的价值
- **玩法**：甜品店开业。第一天订单少，孩子一步步手工做每个蛋糕（打蛋→加糖→烤）。第二天 20 张订单涌进来，手工根本来不及——爆单失败。此时解锁「配方卡」：把做蛋糕的步骤封装成 `做蛋糕(口味, 大小)`，之后每张订单只需一行调用。午高峰的存活压力让孩子自己「发明」函数的理由。改良配方只改一张卡，所有蛋糕同步升级——维护性也玩出来了。
- **魔法时刻**：用 3 张配方卡扛住 50 单午高峰，回看第一天满屏的重复步骤块——「我再也不要复制粘贴了」。
- **进阶**：配方调用配方（做蛋糕调用打奶油）、带返回值的试吃函数、参数默认值（不说口味就是原味）。

### 8. 🐧 排序运动会 · Sorting Olympics — `sort-olympics`
`G5-6` `P0` `Canvas`｜排序算法（冒泡/选择/插入）、算法步骤与比较、效率直觉
- **玩法**：企鹅运动会要按个头排队，但企鹅太胖只能两两比较、相邻交换。孩子先徒手排，计步器记录比较次数；然后「雇佣」算法教练：冒泡教练让高个子一轮轮浮到队尾，选择教练每轮点走最矮的上领奖台，插入教练把新企鹅塞进已排好的队伍。终极环节「算法赛跑」：同一批 30 只企鹅，三条赛道三种算法同时开跑，比较次数实时对决。
- **魔法时刻**：算法赛跑时肉眼看到插入排序在「几乎排好」的队伍上遥遥领先——算法有性格，场景定输赢。
- **进阶**：乱序/倒序/近有序三种赛制、自创混合策略挑战教练、100 只企鹅极限赛（体会 n² 增长）。

### 9. 🗼 二进制灯塔 · Binary Lighthouse — `binary-lighthouse`
`G4-6` `P1` `DOM/Canvas`｜二进制、位值、用 0/1 编码信息
- **玩法**：孩子是海岛灯塔管理员，塔上一排 5 盏灯，每盏只有亮/灭——但这足以对过往船只说出任何数字：灯的位值是 1、2、4、8、16。港口发来订单「向货船报数字 19」，孩子点亮 16+2+1 三盏灯；船回以灯语，孩子解码。学会数字后解锁字母表（A=1…Z=26），开始和幽灵船互通秘密信件。
- **魔法时刻**：发现 5 盏灯居然能表示 0–31 共 32 个数，加一盏灯直接翻倍——指数的力量闪闪发光。
- **进阶**：二进制手指计数彩蛋（一只手数到 31）、灯坏了一盏怎么办（容错引子）、给同学名字编码成灯语串。

### 10. 🖼️ 像素邮局 · Pixel Post Office — `pixel-post`
`G3-5` `P1` `Canvas`｜图像的像素表示、用数字编码图片、行程长度压缩（RLE）
- **玩法**：这家邮局的电报线只能传数字，可客人要寄「图画」！孩子先当发报员：把 8×8 格子画翻译成一串数字发出去；再当收报员：照着收到的数字串涂格子，图案在手里复原——图像=数字，亲手验证。生意做大后电报按字数收费，逼出压缩智慧：「白白白白白」写成「5白」（RLE），同一张图收费砍半。
- **魔法时刻**：收报涂完最后一格，一只完整的像素小猫从数字串里浮现——「原来图片是这样穿过电线的」。
- **进阶**：彩色图（每格 3 个数字，RGB 萌芽）、压缩率排行榜、设计「最难压缩」和「最好压缩」的图各一张（噪点 vs 色块）。

### 11. 🤖 扫地机器人研究所 · Robo-Vacuum Lab — `vacuum-lab`
`G3-4` `P1` `Canvas`｜算法策略、简单规则产生复杂行为、覆盖与效率
- **玩法**：研究所测试扫地机器人。孩子不是遥控它——是给它装「脑子」：从策略芯片库选装（碰壁就随机转向 / 弓字形走 / 沿墙走），按启动后只能观察不能插手，看它在各种户型里表现如何（覆盖率%、耗时、卡住次数三项跑分）。不同户型冠军不同：空旷客厅弓字形碾压，多障碍房间随机反而不容易卡死。
- **魔法时刻**：那台「傻傻乱撞」的随机机器人在迷宫户型里战胜了精心设计的弓字形——简单规则的意外之力。
- **进阶**：自己拼装组合策略（先沿墙一圈再弓字）、设计一个「专克某策略」的户型、双机器人协作分区。

### 12. 🔐 密码社 · Cipher Club — `cipher-club`
`G5-6` `P2` `DOM`｜加密与解密、凯撒密码、替换密码、频率分析入门
- **玩法**：加入课间秘密社团。入社考验：用可旋转的凯撒双转盘给暗号加密、解密。日常任务是传纸条剧情：给好朋友发加密邀请函，截获对手社团的密文并破解（低年级给偏移量提示，高年级用「哪个字母出现最多？大概是 E」的频率分析破解）。终极考核：设计自己的替换密码本，让 AI 破译官来攻，撑过三轮攻击才能当社长。
- **魔法时刻**：没有钥匙，仅靠数字母频率破开一整段密文——像真正的密码学家一样。
- **进阶**：双重加密、空格陷阱（去掉空格防破解）、密码史小剧场（恺撒与二战电报彩蛋）。

### 13. 🐶 机器宠物学校 · Robo-Pet School — `robo-pet-school`
`G5-6` `P2` `Canvas`｜机器学习入门：用例子教而非用规则教、训练数据、误判与纠正
- **玩法**：领养一只出厂时什么都不懂的机器狗。教它「捡球」不是写规则，而是喂例子：把院子里的东西一件件标注「是球 / 不是球」。教得少它就闹笑话（叼回西瓜、番茄、毛线团——因为都是圆的！）；孩子追加「番茄不是球」的例子，它下次就懂了。测验日放它去公园自由捡球，成绩取决于你的教学质量。偏见陷阱关：如果你只教过红色的球，它会对绿网球视而不见——训练数据的偏差被孩子亲身踩到。
- **魔法时刻**：机器狗第一次把没见过的新球正确捡回——它不是记住了，是「学会」了。
- **进阶**：教第二只性格不同的宠物（学得快忘得快）、两人各教一只比赛、观察它「犹豫」的瞬间（置信度可视化）。

### 14. 📦 数据包骑士 · Packet Knights — `packet-knights`
`G5-6` `P2` `Canvas`｜网络与路由：信息拆包传输、最短路径、断线重路由
- **玩法**：王国之间没有一条直达路，信件要拆成编号数据包，由骑士小队分头骑行，经不同城堡中转，到站后按编号拼回原信。孩子当调度官：给每个包选路线，路上桥梁会塌（断线）、某城堡会堵（拥塞），要实时改道；包乱序到达没关系——编号保证拼得回来，丢了哪包就重发哪包。
- **魔法时刻**：主干道塌了，你的包群像水一样自动绕行三条小路，信件依然完整送达——「打不垮的邮政」正是互联网的样子。
- **进阶**：加密包（结合密码社）、抢带宽的双玩家对战、traceroute 彩蛋（看看真实网络上一个包去过哪些城市）。
